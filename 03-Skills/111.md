## 開發流程

| Skill | 用途 |
|-------|------|
| `/tdd` | TDD 紅綠重構流程。要建功能或修 bug 時用，會帶你做 vertical slice（一次一個 test → impl） |
| `/request-refactor-plan` | 產出詳細重構計畫 + 建議 commit 順序，重構前先規劃 |
| `/improve-codebase-architecture` | 掃 codebase 找架構改善機會，參考 CONTEXT.md / ADR，找模組解耦、提升可測試性的切入點 |
| `/triage-issue` | 輸入問題描述，自動查 codebase 找根因，產出 TDD 修復計畫並建 GitHub issue |
| `/github-triage` | 用 label 狀態機管理 GitHub issues，分類、標籤、整理待處理的 bug / feature request |
| `/zoom-out` | 拉高視角，解釋目前這段 code 在整個系統的定位與上下文 |
| `/investigate` | 系統化 debug，四階段：調查、假設、驗證、根因 |
